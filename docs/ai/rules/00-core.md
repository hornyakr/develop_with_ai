# Project contract

- Product: Atlas Workspace, a multi-tenant Next.js App Router training SaaS.
- Use the repository's installed, version-matched documentation before writing framework code. For Next.js 16.2+, inspect `node_modules/next/dist/docs/` first.
- Work from an explicit task brief. Record base commit, writable paths, acceptance criteria, negative tests, and stop conditions.
- One active writer per worktree. Researchers and reviewers are read-only unless a new writer lease is explicitly assigned.
- Keep changes minimal and reviewable. Do not reformat, rename, upgrade dependencies, edit lockfiles, or create migrations outside the task scope.
- Stop on conflicting instructions, unexpected baseline failures, another writer, missing secret, production impact, destructive operation, or a materially larger diff than the brief permits.
